# မြန်မာ ပဋိပက္ခ စောင့်ကြည့်လေ့လာရေးအဖွဲ့ (Myanmar Conflict Observatory - MCO)

ဤစီမံကိန်းသည် မြန်မာနိုင်ငံအတွင်း ဖြစ်ပွားနေသော ပဋိပက္ခဆိုင်ရာ ဒေတာများကို ခွဲခြမ်းစိတ်ဖြာရန်နှင့် မြင်သာသော visualization များဖြင့် တင်ပြပေးနိုင်ရန်အတွက် အသုံးဝင်သော analytical toolkit နှင့် visualization hub တစ်ခုဖြစ်သည်။  
အဓိကရည်ရွယ်ချက်မှာ ရှုပ်ထွေးသော အကြမ်းဒေတာများ (raw data) ကို စနစ်ကျသော အချက်အလက်အမြင်များ (insights) နှင့် အသုံးပြုရလွယ်ကူသော visualization များအဖြစ် ပြောင်းလဲပေးရန် ဖြစ်သည်။

လက်ရှိအသုံးပြုနေသော အဓိကဒေတာကို **ACLED (Armed Conflict Location & Event Data Project)** မှ ရယူထားပြီး၊ ၂၀၂၁ ခုနှစ် ဖေဖော်ဝါရီ ၁ ရက်နေ့ စစ်တပ်အာဏာသိမ်းပြီးနောက်ကာလမှ စတင်သည့် အချိန်ကာလကို အဓိကထား ခွဲခြမ်းစိတ်ဖြာထားသည်။

ဤစီမံကိန်း၏ ရည်ရွယ်ချက်မှာ သုတေသနပညာရှင်များ၊ သတင်းထောက်များ၊ အချက်အလက်ခွဲခြမ်းစိတ်ဖြာသူများအတွက် ပဋိပက္ခဖြစ်ပွားမှုလမ်းကြောင်းများ၊ ပထဝီဝင်အရ အန္တရာယ်မြင့်နေရာများနှင့် ပါဝင်သည့်အဖွဲ့အစည်းများ၏ လှုပ်ရှားမှုများကို ခိုင်မာသော နည်းပညာပိုင်းဆိုင်ရာ မူဘောင်တစ်ခုမှတစ်ဆင့် ရှင်းလင်းစွာ မြင်နိုင်ရန် ပံ့ပိုးပေးခြင်း ဖြစ်သည်။

### စီမံကိန်း အခြေအနေနှင့် ဒေတာ အခြေခံအဆောက်အအုံ (Project Status & Data Infrastructure)

     စတင်သည့်ရက်စွဲ - ၂၀၂၁ ခုနှစ် ဖေဖော်ဝါရီ ၁ ရက် (အာဏာသိမ်းသည့်နေ့)။
     နောက်ဆုံးရက်စွဲ - လက်ရှိအချိန်အထိ (စဉ်ဆက်မပြတ် အပ်ဒိတ်ပြုလုပ်နေသည်)။
     အပ်ဒိတ်ပြုလုပ်ပုံ - PostgreSQL database အတွင်းသို့ အလိုအလျောက် ETL pipeline ဖြင့် ထည့်သွင်းသည်။
     ဒေတာဘေ့စ် - အချက်အလက်များကို ပိုမိုမြန်ဆန်စွာ ရယူနိုင်ရန် PostgreSQL နှင့် SQLAlchemy ORM ကို အသုံးပြုထားသည်။
     တည်ဆောက်ပုံ - စနစ်ကို ပိုမိုကောင်းမွန်စွာ ထိန်းသိမ်းနိုင်ရန် Modular architecture (src, docs, notebooks, scripts) ဖြင့် ပြင်ဆင်ထားသည်။
              
ဤစီမံကိန်းသည် Armed Conflict Location & Event Data Project (ACLED) မှ ဒေတာများကို အသုံးပြုထားသည်။

     Provider - ACLED 
     Location - Myanmar
     Timeframe - Feb 1, 2021 – Present
     လိုင်စင် - ဤ repository အတွင်းရှိ analysis code များသည် open source ဖြစ်သည်။ သို့သော် ACLED data များသည် proprietary ဖြစ်သောကြောင့် raw data files များကို အသုံးပြုလိုပါက ACLED တွင် မှတ်ပုံတင်ရမည်ဖြစ်သည်။ ဤ repository သည် raw data များကို ပြန်လည်ဖြန့်ဝေခြင်း မရှိပါ။
     
### လုပ်ဆောင်နိုင်စွမ်းနှင့် နယ်ပယ် (Possibilities & Scope)

လက်ရှိတွင် အကောင်အထည်ဖော်ထားသော သို့မဟုတ် အဆင့်မြင့် လမ်းပြမြေပုံတွင် ပါဝင်သော အဓိက ခွဲခြမ်းစိတ်ဖြာမှု အစိတ်အပိုင်းများမှာ အောက်ပါအတိုင်း ဖြစ်သည်။

၁။ အချိန်နှင့်အမျှ ပြောင်းလဲမှုများကို ခွဲခြမ်းစိတ်ဖြာခြင်း (Temporal Analysis)

     - ပဋိပက္ခ အကြိမ်အရေအတွက် - နေ့စဉ် / အပတ်စဉ် / လစဉ် ဖြစ်ပွားသော ဖြစ်စဉ်များကို time-series graphs ဖြင့် ပြသခြင်း။
     - သေဆုံးမှု လမ်းကြောင်းများ - အကြမ်းဖက်မှု ပြောင်းလဲမှုများကို သိရှိနိုင်ရန် အချိန်နှင့်အမျှ သေဆုံးမှုနှုန်းများကို ခွဲခြမ်းစိတ်ဖြာခြင်း။
     - ဖြစ်စဉ် အမျိုးအစားများ - တိုက်ပွဲများ၊ အရပ်သားများအပေါ် အကြမ်းဖက်မှုများနှင့် ဆန္ဒပြမှုများ စသည့် အမျိုးအစားများကို ခွဲခြားပြသခြင်း။

၂။ ပထဝီဝင်ဆိုင်ရာ ခွဲခြမ်းစိတ်ဖြာခြင်း (Geospatial Analysis)

     - အန္တရာယ်မြင့်နေရာများ - ပြည်နယ်/တိုင်းနှင့် မြို့နယ်အဆင့် ပဋိပက္ခဖြစ်ပွားသည့်နေရာများကို map ဖြင့် ပြသခြင်း။
     - ပဋိပက္ခ ပျံ့နှံ့မှု - အချိန်နှင့်အမျှ ပဋိပက္ခများ မြန်မာနိုင်ငံအနှံ့ ပျံ့နှံ့လာမှုကို animated visualization များဖြင့် ပြသခြင်း။
     - ဒေသတွင်း ပြင်းထန်မှု - သေဆုံးမှုနှင့် ဖြစ်စဉ်အရေအတွက် အချိုးကို အခြေခံ၍ custom Severity Index ကို တွက်ချက်ပြသခြင်း။

၃။ ပါဝင်သည့်အဖွဲ့အစည်းများ၏ လှုပ်ရှားမှု (Actor Dynamics)

     - အပြန်အလှန် လှုပ်ရှားမှုများ - စစ်ကောင်စီတပ်များ၊ တော်လှန်ရေးအဖွဲ့များ (PDFs) နှင့် တိုင်းရင်းသားလက်နက်ကိုင်များ (EAOs) အကြား ထိတွေ့မှုများကို interactive network graphs များဖြင့် ပြသခြင်း။
     - ဖြစ်စဉ် ပါဝင်မှု - ဖြစ်ရပ်အမျိုးအစားများနှင့် အဖွဲ့အစည်းများ၏ ပါဝင်မှုအခြေအနေများကို Sunburst visualization များဖြင့် ပြသခြင်း။

၄။ အဆင့်မြင့် အချက်အလက်အမြင်များ (Advanced Insights Roadmap)

     အောက်ပါ Natural Language Processing (NLP) အစိတ်အပိုင်းများကို လက်ရှိတွင် ပေါင်းစပ်ထည့်သွင်းနေသည်။
     - Topic Modeling - ACLED ၏ "Notes" များမှတစ်ဆင့် ပဋိပက္ခ၏ အဓိက အကြောင်းအရာများကို အလိုအလျောက် ခွဲခြားသတ်မှတ်ခြင်း။
     - Named Entity Recognition (NER) - ဖြစ်စဉ်မှတ်တမ်းများမှ တပ်ရင်းတပ်ဖွဲ့ အမည်များနှင့် ဒေသခံ လက်နက်ကိုင်အဖွဲ့ အမည်များကို ထုတ်ယူခြင်း။
     - Semantic Search - ဖြစ်စဉ်များ၏ အဓိပ္ပာယ်ကို အခြေခံ၍ ရှာဖွေနိုင်သော vector-based search စနစ် ထည့်သွင်းခြင်း။
     
### စီမံကိန်း ဖွဲ့စည်းပုံ (Project Organization)

စီမံကိန်းကို ပိုမိုစနစ်ကျစေရန် အောက်ပါအတိုင်း ဖွဲ့စည်းထားသည်။

     app.py - အဓိက Streamlit dashboard interface။
     db_manager.py - ဒေတာများကို PostgreSQL အတွင်းသို့ ထည့်သွင်းသည့် pipeline script။
     src/ - အဖွဲ့အစည်းများ ခွဲခြားခြင်းနှင့် data cleaning အတွက် အသုံးပြုသည့် အဓိက logic များ။
     docs/ - သုတေသန စာတမ်းများ၊ အဆိုပြုချက်များနှင့် NLP မဟာဗျူဟာများ။
     notebooks/ - သုတေသနနှင့် အချက်အလက် စမ်းသပ်ခွဲခြမ်းစိတ်ဖြာမှုများ ပြုလုပ်ရာနေရာ။
     scripts/ - ဒေတာ တိကျမှန်ကန်မှု စစ်ဆေးသည့် utility tools များ။

### တာဝန်ယူမှု၊ တာဝန်ခံမှုနှင့် ကျင့်ဝတ် (Disclaimer & Ethics)

     ဤစီမံကိန်းတွင် ခွဲခြမ်းစိတ်ဖြာထားသော ဒေတာများသည် အမှန်တကယ်ဖြစ်ပွားနေသော အကြမ်းဖက်မှုများနှင့် လူမှုဒုက္ခများနှင့် ဆက်စပ်နေသည်။ ဤစီမံကိန်း၏ ရည်ရွယ်ချက်မှာ သုတေသနအတွက် objective clarity ပေးရန် ဖြစ်သည်။
     ACLED data များသည် "Verified Floor" (အတည်ပြုပြီး အခြေခံကိန်းဂဏန်း) ကို ကိုယ်စားပြုပြီး၊ အတည်ပြုနိုင်သော အနည်းဆုံး သေဆုံးမှုနှုန်းများကိုသာ ဖော်ပြထားခြင်း ဖြစ်သည်။
     Visualization များ၏ တိကျမှုသည် မူလ data source အပေါ် မူတည်ပါသည်။ မူလဒေတာများအတွက် အခွင့်အရေးအားလုံးသည် ACLED ထံတွင်သာ ရှိပါသည်။
